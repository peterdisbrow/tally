v3 Implementation Phases

Ordered by impact. Each phase ships independently — you get wins along the way, not all at the end.

Phase 1 — UI Overhaul + Visual Organization
Highest Impact
~2 weeks

The single biggest gap. Rundown Studio wins users because it looks modern and feels fast. We need to match that bar while keeping our power-user features.

  • 1
    Redesigned rundown editor — Clean card-based layout with proper spacing, subtle shadows, smooth transitions. Each item row gets a left color stripe for visual categorization.
  • 2
    Item color coding — Color picker per item or section header. Pre-set palette (worship=blue, message=purple, transition=gray, media=orange, prayer=green) plus custom hex. Colors carry through to all output views.
  • 3
    Section headers redesign — Collapsible section groups with color bands, item counts, and cumulative duration. Drag entire sections as a unit.
  • 4
    Keyboard shortcuts for live show — Spacebar or Right Arrow = GO, Left Arrow = Back, Escape = End Show confirm, N = toggle Next Up panel, T = toggle theme, F = fullscreen. Show a shortcuts overlay on "?".
  • 5
    Smooth drag-and-drop — Replace current drag implementation with a polished version: spring animations, clear drop zones, drag preview card, haptic-style visual feedback.
  • 6
    Church service templates — Ship 6 built-in templates: Standard Sunday, Contemporary Worship, Traditional/Liturgical, Christmas Eve, Easter Sunday, Youth Service. Each pre-populated with typical items, durations, and column structure.
  • 7
    Empty state redesign — When no plans exist, show a beautiful onboarding card: "Create your first rundown" with template previews and a quick-start wizard.
  • 8
    Inline duration editor — Click duration to get a smart picker (not just a text field): preset buttons (0:30, 1:00, 2:00, 5:00, 10:00, 15:00, 30:00) plus manual entry. Shows running total for the service.
Why this matters: In user testing, Rundown Studio converts users in under 2 minutes because the UI is self-explanatory. Our current UI requires a learning curve that loses casual volunteers.
Phase 2 — Smart Timing + New Output Views
High Impact
~2 weeks

This is Rundown Studio's killer feature. When a worship set runs 3 minutes long, every downstream item should auto-adjust. Plus we need dedicated output views that competitors offer.

  • 1
    Smart cue timing engine — Track planned vs. actual duration per item. When an item runs over/under, cascade the delta to all subsequent items' start times in real-time. Show +/- variance inline on each row. NEW
  • 2
    Over/under indicator — Each item in show mode shows a colored badge: green (on time), yellow (within 1 min), red (over by 1+ min). The current item's timer changes color as it approaches zero.
  • 3
    Projected end time — Show "Service ends at 11:47 AM (+3 min late)" in the show mode header, updating in real-time as items run over/under. NEW
  • 4
    Confidence monitor view — New output at /rundown/confidence/:token. Large current-item display with timer, next 3 items preview, over/under status, and projected end time. Designed for stage-side monitors. NEW
  • 5
    Studio clock view — New output at /rundown/clock/:token. Large digital clock with current item countdown underneath, minimal chrome. Optionally shows next item. Designed for wall-mounted displays. NEW
  • 6
    Enhanced speaker timer — Add "time remaining in service" below the item timer. Add a progress ring animation. Color transitions: green → yellow → red → flashing red (overtime).
  • 7
    Show mode redesign — Bigger, bolder current cue. Redesigned Next Up panel with color stripes and over/under badges. Floating GO/Back buttons always visible. Show elapsed service time prominently.
  • 8
    Timer color theming — All timer views (speaker, confidence, studio clock) support custom color themes: standard dark, standard light, red/green (for cameras), and custom brand colors.
Rundown Studio's edge: Their smart timing is why production teams choose them. No other tool auto-cascades timing changes. We build this and we're the first church-focused tool with it.
Phase 3 — Workflow Power Features
Medium Impact
~2 weeks

Quality-of-life features that make the daily workflow faster. These are the things that make TDs stay on the platform once they've tried it.

  • 1
    Per-item checklists — Add a checklist to any item (e.g., "Mic check", "Slides loaded", "Video cued"). Checkboxes sync in real-time across collaborators. Show completion percentage on each item row. NEW
  • 2
    PDF export — Generate a clean, print-ready PDF of the rundown. Include: church logo, service date, all items with durations, custom columns, color coding, notes. Option for "tech team" (all columns) vs "band" (music items only) vs "pastor" (message + notes). NEW
  • 3
    CSV import/export — Import from spreadsheet (map columns on import). Export to CSV for use in other tools. Support Planning Center CSV format. NEW
  • 4
    Undo/redo — Client-side undo stack for the last 20 actions (add, edit, delete, reorder). Ctrl/Cmd+Z to undo, Ctrl/Cmd+Shift+Z to redo. Visual toast showing "Undid: Delete item 'Worship Set'". NEW
  • 5
    Duplicate item — Right-click or action menu to duplicate an item (with all metadata, notes, attachments). Also duplicate section (all items within). NEW
  • 6
    Item notes markdown — Support basic markdown in notes (bold, italic, lists, links). Render preview in the notes panel and in output views.
  • 7
    Quick-add from keyboard — Press "A" to add a new item at the bottom, or "I" to insert below current selection. Tab through fields (title → duration → assignee → notes → done). Enter to save and start next item.
  • 8
    Plan comparison — Side-by-side diff of two plans (this week vs last week). Highlight what changed, what was added/removed. Great for reviewing plan changes before going live. NEW
  • 9
    Notification preferences — Collaborators can opt into email/push notifications for: plan changes, going live, show ended, assignment changes. NEW
Phase 4 — Teleprompter + Script System
High Impact
~1.5 weeks

Rundown Creator's strongest feature. Pastors and worship leaders need scripts — song lyrics, sermon notes, liturgy, announcements. Building this in means one less tool in the workflow.

  • 1
    Script editor per item — Rich text editor panel that opens alongside the item. Support headings, bold/italic, highlight, and speaker labels. Separate from "notes" — scripts are for on-screen display, notes are for the tech team. NEW
  • 2
    Teleprompter output — New view at /rundown/prompter/:token. Large scrolling text, auto-scroll at configurable speed, manual scroll override. Current item's script highlighted, next item previewed below. Dark background, high-contrast text. NEW
  • 3
    Auto-advance script — When the operator advances to the next cue, the teleprompter automatically scrolls to the next item's script. No manual intervention needed.
  • 4
    Lyrics integration — If an item is tagged as "Worship/Song", offer to pull lyrics from a built-in search (public domain hymns + user-uploaded lyrics library). NEW
  • 5
    Announcement cards — Rich announcement templates that can be assigned to items. Include title, body, image, and display duration. Outputtable to a separate "announcements" display view. NEW
Rundown Creator's edge: Their built-in teleprompter is why broadcast teams choose them over Rundown Studio. We build this church-optimized (with lyrics + liturgy support) and we beat both.
Phase 5 — Integrations + Polish
Polish
~1.5 weeks

Final layer. These are the features that make reviewers write "wow, they thought of everything."

  • 1
    OSC protocol support — Send/receive OSC messages for show control integration with lighting boards (ETC Eos, GrandMA), audio consoles, and custom systems. Map OSC triggers per cue item. NEW
  • 2
    Drag-and-drop media — Drag images, PDFs, or videos directly into item rows from the desktop. Auto-attach and show thumbnail preview inline.
  • 3
    Multi-week calendar view — See plans on a calendar grid (week/month view). Quickly spot gaps, copy plans to future dates, see which rooms have plans scheduled.
  • 4
    Analytics dashboard — Service timing trends over weeks/months. Average overtime per item type. Most-used templates. Team utilization. Which items consistently run over? NEW
  • 5
    Webhook support — Fire webhooks on key events (show started, show ended, cue advanced). Let churches integrate with their own systems (automation, logging, notifications). NEW
  • 6
    Offline mode — Cache the current plan locally. If internet drops during a live show, keep running from cache. Sync changes when reconnected. NEW
  • 7
    Accessibility audit — WCAG 2.1 AA compliance across all views. Screen reader support, keyboard navigation, high contrast mode, focus indicators.
  • 8
    Performance optimization — Lazy-load output views. Optimize WebSocket payload sizes. Add service worker for caching static assets. Target sub-1s initial load on show mode.

Stack Recommendation

Stay vanilla HTML/JS/CSS — but modernize the patterns

The current stack works. A React migration would burn 3+ weeks on infrastructure with zero user-visible improvement. Instead:

  • -
    Adopt a component pattern — Each UI element (item row, section header, modal, timer display) becomes a self-contained render function with its own state. No framework needed.
  • -
    CSS custom properties for theming — All colors, spacing, and sizes as CSS variables. Switch themes by changing one class on the body.
  • -
    CSS animations for polish — Transitions on drag, status changes, timer color shifts. Pure CSS, no library.
  • -
    Keep server-rendered output views — Show mode, timer, confidence monitor stay as standalone HTML pages. Fast initial load, no JS framework overhead, works on any device.
